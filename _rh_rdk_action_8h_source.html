4 /** \class CRhRdkAction
5 
6  This class allows RDK clients to add their own custom menus (and possible future tool buttons)
7  to the RDK content editor dialog. Each action is identified by a UUID. Actions handle the functionality
8  of a UI item (menu item or future tool button); UI items refer to the action that they represent. This
9  allows the same action to be carried out from different UI items without implementing the action in each place.
10  An action can be enabled or disabled, checked or unchecked; this is reflected in the associated UI item.
11  Actions also have a caption, an icon and a tooltip (not currently used). When the associated UI item
12  is clicked by the user, if the action is enabled its Execute() method is called. Actions can be grouped
13  by setting their group value. These cause their associated UI items to act like radio buttons. Group zero
14  is the default and means that the action is not in a group.
15 
16 To use this feature clients should override the relevant methods in CRhRdkPlugIn.
17 
18  \see IRhRdkMenu
19  \see CRhRdkPlugIn::AddCustomEditorActions()
20  \see CRhRdkPlugIn::UpdateCustomEditorActions()
21  \see CRhRdkPlugIn::AddCustomEditorMenu()
22 */
